# Track Spec: OPG/OBG Markdown Report Quality Improvements (v1)

## Background
Early report outputs were technically correct but hard to compare across scenarios and categories. This track standardized report structure, terminology, and evidence presentation.

## Objective
Improve report clarity and actionability without changing underlying model logic.

## Scope
- Table structure and sorting consistency
- Direction-neutral evidence language
- Explicit scenario labeling and assumptions
- Clear separation of discretionary vs non-discretionary categories where relevant

## Acceptance Criteria
- Reports are readable without code context.
- Key decisions (increase/decrease/maintain) are traceable to evidence rows.
- Generated markdown is stable enough for downstream publication.
